使用 Github Pages 和 Hexo 建立个人博客

发表于 2017-01-11 | 分类于 其他 |

Github Pages 是什么

由用户编写的，托管在 Github 上的静态网页， Github 推出 Pages 功能，本意是允许用户自定义项目首页，用来替代默认的源码列表，但是也有很多人用它来托管自己的个人博客。

Hexo 是什么

Hexo 是一个快速、简洁且高效的博客框架。

安装 Git

  • Git 下载地址
    选择对应版本，下载后安装即可

  • 配置用户名与邮箱

  • 配置 Github SSH Key
    使用 SSH Key 可以使我们 push 更新到 Github 时无需输入用户名与密码。

    • 生成公钥与私钥

    • 添加到 Github Settings
      打开 id_rsa.pub，复制里面的内容。
      登陆 Github ，点击右上角 Settings ，点击 SSH And GPG keys ，New SSH Key ，将复制的内容粘贴在 Key 中 ，点击 Add SSH Key。

    • 测试 SSH 是否配置成功

安装 Node.js

Node.js下载地址 下载安装即可。

安装 Hexo

创建网站

这时候已经可以用 hexo s 命令进行本地预览网页，访问地址 http://localhost:4000

修改配置

主要是站点根目录下的 _config.yml 文件，找到 deploy 部分。

修改为：

repository 部分需要修改为个人在 Github 上创建的仓库地址，名称必须为 username.github.io ， username 替换为 Github 用户名。

部署到Github

  • 首次需要安装 hexo-deployer-git

  • 生成静态网页

  • 本地预览网页

  • 推送到 Github

Hexo 主题

通过修改 _config.yml 文件中的 theme 字段可以更换主题。
本博客使用的是 NexT 主题 ，该主题作者提供了完善的配置文档说明，详细配置可自行查看。

写作命令

  • 新建文章

    执行后会在 source/_post 目录中生成一个 .md 格式的文件，直接打开使用 markdown 语法进行写作即可。

  • 更新站点
    文章编辑完成后，通过以下命令完成更新与发布。
